= Querying the Knowledge Base
:navtitle: Querying the Knowledge Base

This page describes what good query behaviour looks like and how to validate that your knowledge base is returning relevant results in both Chat and Agent modes.

== Effective Query Strategies

* *Be specific* — narrow queries return higher-quality chunks. Instead of "Tell me about HR", ask "What are the steps to request parental leave?"
* *Use natural language* — the embedding model (`all-MiniLM-L6-v2`) is optimized for sentence-level semantic similarity.
* *Iterate* — if the first response lacks detail, rephrase or narrow the question and resubmit.

== Expected Results in Chat Mode

* Relevant excerpts from the correct document(s) are cited alongside the answer.
* For factual, policy-based questions the answer should closely mirror the source document language.
* If no relevant documents are found, the model will indicate it does not have enough context rather than hallucinating an answer.

== Expected Results in Agent Mode

* The agent may issue *multiple retrieval calls* before composing the final answer — this is expected behaviour for complex queries.
* Each tool call and its result are visible in the reasoning trace displayed in the UI.
* The synthesized answer covers all relevant sub-topics and cites the source chunks used at each retrieval step.

== Validating Retrieval Quality

To confirm the knowledge base is returning accurate results:

. Ask a question whose answer appears verbatim in one of your uploaded documents.
. Check that the cited source matches the expected document.
. If results are poor, consider:
** Re-ingesting the document with smaller chunk sizes
** Increasing the top-k retrieval count in the sidebar
** Reviewing the document format (scanned PDFs may require OCR pre-processing)
