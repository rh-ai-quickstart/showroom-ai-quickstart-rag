= Accessing the Cluster
:navtitle: Accessing the Cluster
:imagesdir: ../images

All RAG components run inside the `llama-stack-rag` namespace on OpenShift.
This page walks through what you should see for pods, services, and routes — and how to reach the chatbot UI.

== Accessing the Cluster
From the OpenShift Demo Platform click advanced settings. In the advanced settings is a username and password to login to the cluster.

image::advanced-settings.png[Advanced settings]

Next we need to login to the cluster via the OpenShift UI. In the demo platform click the OpenShift UI button at the top right.

image::openshift-click.png[OpenShift UI]

Login with the username and password from the advanced settings.

== Checking Pods

After logging in to the OpenShift UI, we need to check the pods in the namespace. You will see a single project in the UI. In the top left hand corner click on the hamburger menu and select "Workloads/Pods".

image::pods.png[Pods]

A healthy cluster will show output similar to the photo above.

.Key pods to verify
[cols="2,3",options="header"]
|===
|Pod name contains |Purpose

|`llamastack-ui`
|Streamlit chat interface served to the user

|`llamastack`
|LlamaStack server — orchestrates the LLM, embeddings, and tools

|`predictor` (LLM)
|KServe / vLLM model server for the generation model (e.g., `Llama-3.2-3B-Instruct`)

|`predictor` (safety)
|KServe / vLLM model server for the guard/shield model (e.g., `Llama-Guard-3-8B`) — only present if safety was enabled

|`postgresql`
|PostgreSQL + PGVector — stores document embeddings for the vector store

|`rag-ingest-pipeline`
|Kubeflow Pipeline job that ingested the initial FantaCo documents — `Completed` is the expected final state
|===

[NOTE]
====
The predictor pods show `2/2` ready in the default configuration.
If the system was configured with `RAW_DEPLOYMENT=false`, expect `3/3`.
The `llamastack` pod will not reach `Running` until all predictor pods are healthy.
====

=== Using the CLI to check pods

In order to run commands from the CLI, we need to login to the cluster via the CLI. Navigate to the OpenShift UI and click the button in the top right corner that says "Copy Login Command".

image::login-command.png[Copy Login Command]

Paste the command into the terminal and press enter.

Now we can run commands to check the pods.

[source,bash]
----
oc get pods -l component=predictor -n llama-stack-rag
----

To check the LlamaStack orchestration pod specifically:

[source,bash]
----
oc get pods -l app.kubernetes.io/name=llamastack -n llama-stack-rag
----

== Checking Services

Services expose each component internally within the cluster:

In the OpenShift UI, click on the "Networking/Services" menu item in the left hand navigation.

image::services.png[Services]

Or run the following command from the CLI:

[source,bash]
----
oc get svc -n llama-stack-rag
----

.What each service does
[cols="2,3",options="header"]
|===
|Service |Purpose

|`llamastack`
|Internal REST API for the LlamaStack server (port 5000)

|`llamastack-ui`
|Internal HTTP endpoint for the Streamlit UI (port 8501)

|`postgresql`
|Internal PostgreSQL + PGVector endpoint (port 5432)
|===

== Checking Routes

Routes expose the UI and API to the outside world. In the OpenShift UI, click on the "Networking/Routes" menu item in the left hand navigation.

image::routes.png[Routes]

Or run the following command from the CLI:

[source,bash]
----
oc get routes -n llama-stack-rag
----

Expected output:

[source]
----
NAME              HOST/PORT                                                    PATH   SERVICES        PORT    TERMINATION   WILDCARD
llamastack        llamastack-llama-stack-rag.apps.<cluster-domain>                    llamastack      5000    edge          None
llamastack-ui     llamastack-ui-llama-stack-rag.apps.<cluster-domain>                llamastack-ui   8501    edge          None
----

.Route breakdown
[cols="2,3",options="header"]
|===
|Route |Purpose

|`llamastack-ui`
|*This is the URL you open in your browser.* It serves the Streamlit chat interface over HTTPS.

|`llamastack`
|The LlamaStack REST API — useful for direct API access, scripting, or connecting external clients.
|===

Open the `HOST/PORT` value for `llamastack-ui` in your browser to launch the chat interface.

== What to Expect in the UI

When the Streamlit UI loads you will see:

* A chat input box at the bottom of the screen for submitting questions
* A left sidebar with configuration options:
** *Knowledge Base* — select which vector store collection to query
** *Mode* — toggle between *Chat* (simple RAG) and *Agent* (agentic RAG)
** *Sampling parameters* — adjust temperature, top-p, and max tokens
** *System prompt* — override the default system prompt

The application comes pre-loaded with FantaCo's internal HR, procurement, sales, and IT documents so you can begin asking questions immediately.
